# 常见场景

展示如何在日常中使用 BYOAO 的真实场景。每个场景在上下文中介绍相关技能。

---

## 1. 第一次 Weave — 连接你的笔记

**时机：** 刚创建知识库，已有一些笔记（导入的或新写的）。

**步骤：**

1. 确保知识库在 Obsidian 中打开且 CLI 已启用
2. 打开 Agent Client 面板
3. 运行 `/weave`
4. 审查建议的改动 — frontmatter 添加、wikilinks、Glossary 术语
5. 确认并应用
6. 按 `Cmd+G` 查看图谱

**预期结果：**
- 笔记获得 frontmatter：`title`、`date`、`type`、`domain`、`tags`、`references`
- `date` 始终有值 — 从内容推断，或以文件创建时间为兜底
- 来自云端的笔记会获得 `source` 字段链接回原文
- 人名/项目名的纯文本提及变为 `[[wikilinks]]`
- 反复出现的概念被建议加入 Glossary（5+ 次自动建议，3+ 次向用户确认）
- 摘要显示："Enriched 23 files, added 87 wikilinks, 5 new Glossary terms"
- 如果目录结构混乱，/weave 会建议运行 `/organize`

**小贴士：**
- 先在小范围运行（用 `folder=Daily/`）看看效果
- 在 `.byoao/backups/` 查看备份，需要时可以撤销
- 随时重新运行 /weave — 它是幂等的（不会重复已有链接）
- /weave 之后，可以运行 `/organize` 基于增强的元数据重新整理目录

---

## 2. Weave 后整理 — 重新组织知识库

**时机：** 已运行 `/weave`，笔记有了 frontmatter，但文件散落在混乱的目录中（采纳已有知识库时常见）。

**步骤：**

1. 确保已运行 `/weave`（agent 需要 `type` 元数据来决定文件归属）
2. 运行 `/organize` 查看目录重组建议

```
/organize                    # 分析整个知识库
/organize dry-run            # 预览变更但不执行
/organize scope=Knowledge/   # 聚焦特定文件夹
```

3. 审查前后对比摘要 — agent 按操作分组（如 "将 12 个会议笔记移到 Meetings/"）
4. 批准或调整 — 可以全部接受、全部拒绝或逐个选择
5. Agent 使用 `obsidian move` 执行每次移动，自动更新所有 wikilinks

**预期结果：**
- 文件按类型整理：日记 → `Daily/`、会议 → `Meetings/`、参考 → `Knowledge/`
- 完整的文件组保持在一起（如 sprint 文件夹中的关联文件不会被拆散）
- 所有 wikilinks 和反向链接自动更新 — 无断裂引用
- 移动后验证图谱健康

**小贴士：**
- 先用 `dry-run` 看看会发生什么变化
- 用 `scope=` 逐个文件夹整理，掌控更精细
- Agent 不会在收益不明确时移动文件 — 默认保守策略

## 3. 每周回顾 — 保持图谱更新

**时机：** 写了一周的笔记，想整合到图谱中。

**步骤：**

1. 运行 `/weave` 连接本周笔记
2. 运行 `/diagnose` 检查知识库健康

```
/weave
/diagnose
```

**/diagnose 报告什么：**
- 没有 frontmatter 的笔记（新的、未处理的）
- 孤立笔记（没有连接任何内容）
- 断裂的 wikilinks（链接到不存在的笔记）
- AGENTS.md 漂移（引用了不存在的笔记）

**后续行动：**
- 孤立笔记：决定是否需要连接或归档
- 断裂链接：创建缺失的笔记或修复链接
- 缺少 frontmatter：下次 /weave 运行时会处理

## 4. 追踪想法 — 我的思维如何演变的？

**时机：** 想了解某个概念在笔记中随时间的发展轨迹。

**例子：** 你几个月来一直在思考 "API 限流"，想看完整脉络。

**步骤：**

```
/trace topic="rate limiting"
```

**你会得到：**
- 提及 "rate limiting" 的每条笔记的时间线
- 识别出的阶段：发现 → 调研 → 决策 → 实施
- 标记的转折点："在阅读 [[System Design Doc]] 后，你的方法发生了变化"
- 未闭合的线索："你探索了令牌桶方案但六月后没有后续"

**适用场景：**
- 做决策前 — 看看你已经探索过什么
- 写复盘时 — 追踪项目的演变
- 帮新人了解 — "这个决策是这样做出的"

## 5. 发现隐藏模式 — 我遗漏了什么？

**时机：** 知识库有 50+ 条笔记，想看全局。

**步骤：**

```
/emerge
```

**你会得到：**
- **反复出现的未回答问题** — "你在 4 条笔记中问过'迁移时间线是什么'但从未回答"
- **隐含的决策** — "笔记在四月左右从方案 A 转向方案 B，但没有决策记录"
- **被遗忘的线索** — "'数据网格' 在 1-3 月的 6 条笔记中出现，之后再未提及"
- **跨领域连接** — "你的'支付'和'认证'笔记都引用了'限流'但彼此没有链接"
- **结构观察** — 连接最多的笔记、最活跃的领域、可能的盲区

**适用场景：**
- 每月 — 作为 "知识库回顾"
- 新季度开始时 — 上季度涌现了什么模式？
- 感觉卡住时 — 让知识库展示你一直在围绕什么打转

**小贴士：**
- 用 `scope=Projects/` 聚焦特定区域
- 用 `depth=deep` 做深度分析（读取每条笔记，耗时更长）

## 6. 桥接两个主题 — 发现隐藏联系

**时机：** 怀疑两个主题有关联但说不清楚。

**例子：** 你分别在做 "支付处理" 和 "用户引导"。它们在知识库中有联系吗？

**步骤：**

```
/connect from="payment processing" to="user onboarding"
```

**你会得到：**
- **共同笔记** — 同时讨论两个主题的笔记
- **共同人物** — 参与两个领域的人
- **图谱路径** — 两个主题如何通过中间笔记连接（最多 3 跳）
- **强度评估** — 强、中、弱连接
- **建议操作** — 具体要链接的笔记、要创建的 hub notes

**适用场景：**
- 跨团队协作 — "我的工作和你的有什么联系？"
- 战略思考 — "这些项目的共同点是什么？"
- 写作 — "我需要解释 A 和 B 的关系"

**如果没找到连接：** 这也是有用的结果。说明这两个主题在你的笔记中还没有交集 — 也许应该有，也许不需要。

## 7. 生成想法 — 接下来该做什么？

**时机：** 知识库已有大量内容，想获得有创意、可行动的建议。

**步骤：**

```
/ideas
/ideas focus="infrastructure"   # 聚焦特定领域
```

**你会得到：**
- **综合想法** — 将两条已有线索组合成新东西
- **缺口想法** — 知识库暗示需要但尚不存在的东西
- **连接想法** — 两个应该沟通的人/项目
- **放大想法** — 将小想法扩大规模
- **质疑想法** — 挑战知识库中默认的假设

每个想法引用 2+ 条笔记，并附具体下一步。

**适用场景：**
- 季度规划 — "我的知识库揭示了什么机会？"
- 感觉卡住时 — 让知识库建议该做什么
- 跨领域创新 — 找到不同领域的交集

---

## 8. 质疑信念 — 我的想法对吗？

**时机：** 要做重大决策，想用自己的历史来验证。

**例子：** 你认为 "我们应该迁移到微服务架构"。

**步骤：**

```
/challenge belief="我们应该迁移到微服务架构"
```

**你会得到：**
- 笔记中的支持证据（先公平评估）
- 反证：矛盾、立场随时间的变化
- 信念依赖的未陈述假设
- 缺失的视角（谁的观点不在？）
- 置信评级：强 / 中 / 弱 / 被反驳

**适用场景：**
- 重大决策前 — "我的知识库支持这个吗？"
- 解决分歧 — "我自己的笔记怎么说的？"
- 复盘 — "当时的选择对吗？"

---

## 9. 检测漂移 — 我做到了我说要做的吗？

**时机：** 想对比计划和实际执行。

**步骤：**

```
/drift                         # 最近 30 天
/drift period=60d              # 最近 60 天
/drift focus="API 迁移"        # 聚焦特定项目
```

**你会得到：**
- 每个意图被分类为：对齐、延迟、漂移、放弃或涌现
- 模式检测：优先级置换、范围蔓延、目标放弃
- 涌现工作被识别（未计划但重要的）
- 需要思考的问题（描述性的，不带评判）

**适用场景：**
- Sprint/月末 — "我执行了吗？"
- 季度回顾 — "我的精力实际去了哪里？"
- 感觉不堪重负时 — "什么在消耗我的注意力？"

## 建立节奏

以下节奏效果不错：

| 频率 | 操作 | 技能 |
|------|------|------|
| 每天 | 写日记，记录会议和想法 | — |
| 每周 | 连接新笔记，检查健康 | `/weave` + `/diagnose` |
| weave 后 | 按需整理目录结构 | `/organize` |
| 好奇时 | 追踪某个主题的演变 | `/trace` |
| 每月 | 在知识库中寻找模式 | `/emerge` |
| 每季度 | 生成想法，回顾漂移 | `/ideas` + `/drift` |
| 重大决策前 | 压力测试假设 | `/challenge` |
| 需要时 | 桥接正在做的两个主题 | `/connect` |

目标不是每天运行所有技能。自由写作，每周编织，每月思考，每季度反思。
